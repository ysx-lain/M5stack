# 🤖 M5Stack CoreS3 语音对话机器人 - Coding Plan Lite
> 轻量级本地LLM语音交互系统

---

## 📋 项目概述
- **硬件**：M5Stack CoreS3 + LLM模块 + 可选ESP32S3节点
- **功能**：语音对话 + 网页端交互 + 传感器交互 + 多设备控制
- **架构**：CoreS3作为网关 + 本地LLM计算 + 局域网服务
- **代码量**：~2000行
- **开发周期**：3天

## 🧩 软件模块设计
### 1. 核心模块
- `main.cpp`：主程序入口、状态管理
- `ui_manager.h`：UI界面管理、多APP切换
- `llm_client.h`：LLM模块通信封装
- `web_server.h`：局域网Web服务
- `sensor_manager.h`：传感器数据处理
- `device_controller.h`：外部设备控制

### 2. APP列表
1. **对话APP**：语音/文字对话界面
2. **控制APP**：ESP32S3设备控制界面
3. **表情APP**：表情展示与交互
4. **设置APP**：网络、参数配置

### 3. 网页端
- 对话界面：支持文字/语音输入
- 设备管理：查看和控制连接的ESP32S3
- 系统设置：参数配置、固件更新

## 📝 API接口设计
### LLM模块接口
```c++
// 语音识别
String asrRecord(int timeout = 5000);
// 文本对话
String llmChat(String prompt, bool stream = true);
// 语音合成
bool ttsSpeak(String text);
```

### Web接口
- `GET /`：网页主界面
- `GET /api/chat?text=xxx`：对话接口
- `WebSocket /ws`：实时消息推送
- `POST /api/control`：设备控制接口

### 串口控制协议
```
> AT+DEVICE=ESP32S3-1
> AT+SEND=Hello
< OK: Message sent
```

## 🔧 编译与部署
### 编译环境
- Arduino IDE 2.0+
- M5Stack板级支持包 2.0.5+
- 依赖库：M5CoreS3, WebServer, WebSockets, ArduinoJson

### 部署步骤
1. 配置WiFi信息
2. 配置LLM模块串口参数
3. 编译上传到CoreS3
4. 访问设备IP地址使用网页端
5. 如需控制ESP32S3，连接对应串口

## 🚨 风险与应对
| 风险 | 应对方案 |
|------|----------|
| LLM模块通信不稳定 | 增加超时重传机制、错误校验 |
| 语音识别准确率低 | 增加降噪处理、端点检测 |
| 局域网连接不稳定 | 自动重连、离线模式支持 |
| 内存不足 | 优化对话历史长度、使用PSRAM |
