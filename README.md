# M5Stack CoreS3 LLM 机器人

## 功能特点

### 第一步已完成：初始化界面
- ✅ **WiFi配置持久化** - 使用ESP32 Preferences存储，配置一次后开机自动连接，不需要重新配置
- ✅ **设备状态检测** - 开机自动检测IMU传感器和LLM Module连接状态
- ✅ **白色背景UI** - 清爽的浅色风格，支持中文显示
- ✅ **加载动画** - 启动时有旋转加载动画，进度可视化
- ✅ **点击动画效果** - 点击图标时有缩放动画反馈，交互体验好
- ✅ **预留蓝牙接口** - 已预留蓝牙设备管理代码结构，后续可扩展
- ✅ **低资源占用** - 使用FreeRTOS双核处理，LLM推理在核心0，UI在核心1，不卡顿
- ✅ **AP配网** - 未配置WiFi时自动开启AP，通过浏览器配置

## 启动流程

1. **开机启动动画** - 旋转加载动画显示"初始化中"
2. **加载配置** - 从Flash读取已保存的WiFi配置
3. **检测传感器** - 检查IMU是否正常
4. **连接WiFi** -
   - 有配置：自动连接，显示连接进度动画
   - 无配置：开启AP `M5-LLM-Setup`，访问 `192.168.4.1` 配置
5. **检测LLM Module** - 检查连接，初始化模型
6. **显示状态汇总** - 显示各模块连接状态
7. **点击进入主界面** - 桌面2x2图标布局，支持滑动切换

## 主界面功能

- **聊天** - 语音对话，点击麦克风说话，ASR+LLM+TTS全流程
- **控制** - WiFi重连按钮，蓝牙扫描预留位置
- **表情** - 摇晃CoreS3切换表情，演示IMU使用
- **设置** - 查看各模块状态

## 设计特点

### 内存占用低
- 使用FreeRTOS任务分离，LLM在核心0，UI在核心1
- 合理限制聊天历史长度，避免内存溢出
- 使用Preferences存储配置，占用空间极小

### 动画交互
- 点击图标时缩放松放动画
- 启动旋转加载动画
- 滑动切换页面
- 点击音效反馈

### 持久化配置
- WiFi配置保存在Flash，重启不丢失
- 无需每次开机重新输入

### 蓝牙预留
- 已定义`Device`结构
- `bluetooth_enabled`标志位
- 控制页预留蓝牙扫描按钮位置

## 编译运行

使用PlatformIO，打开项目目录，选择`m5stack-cores3`环境编译上传：

```bash
platformio run --target upload
```

## 硬件连接

- LLM Module → CoreS3 PortC
  - RX: 18
  - TX: 17
-  Baud rate: 初始115200 → 提升到1500000
